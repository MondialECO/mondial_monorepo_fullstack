CRITICAL BUG FIXED:
─────────────────
❌ User types in form fields
   User selects dropdown value
   ❌ Previously typed fields get RESET

ROOT CAUSE:
──────────
useEffect dependency array included `form` and `getPhaseData`
→ Form.reset() called repeatedly on every dropdown change
→ User input from typing lost

EXACT ROOT CAUSE:
─────────────────
File: src/hooks/usePhase2Step1Form.ts, line 72

When user selected dropdown:
1. form.setValue() updates form state
2. Component re-renders
3. getPhaseData or form dependency changes
4. Effect runs: form.reset(savedData)
5. savedData only has SAVED values (not user's current typing)
6. form.reset() overwrites form with saved data
7. User input LOST ❌

FIX APPLIED:
────────────

FIX 1: Remove Problematic Dependencies
File: src/hooks/usePhase2Step1Form.ts (line 72)

BEFORE: }, [progress, initialData, form, getPhaseData]);
AFTER:  }, [progress, initialData]);

Why:
- `form` is stable (doesn't need to be dependency)
- `getPhaseData` gets recreated on parent re-render (false positive trigger)
- Effect now only runs when progress or initialData actually change
- form.reset() no longer called on dropdown changes
- User input preserved ✓

FIX 2: Add Missing Import
File: src/components/entrepreneur/LegalIdentityForm.tsx (line 4)

BEFORE: import { Lightbulb } from 'lucide-react';
AFTER:  import { Lightbulb, ChevronDown } from 'lucide-react';

Why:
- ChevronDown icon was used but not imported
- Caused runtime error
