CRITICAL BUG FIXED:
─────────────────
❌ Users could NOT access /dashboard/entrepreneur/phase-2/step-2
   - Even after completing step-1
   - RouteGuard would redirect them back to step-1
   - Infinite redirect loop prevented flow progression

ROOT CAUSE:
──────────
Custom hook useEntrepreneurProgress() created SEPARATE state instances
for each component instead of sharing state:

  Phase2Step1Client: calls useEntrepreneurProgress() → State Instance #1
  Phase2Step2Page:   calls useEntrepreneurProgress() → State Instance #2
  RouteGuard:        calls useEntrepreneurProgress() → State Instance #3

When Phase2Step1Client updated state #1, Phase2Step2Page couldn't see those
updates (reading from #2), so RouteGuard would see currentStep=1 and redirect.

THE FIX:
────────
✅ Created Context-based provider to ensure ONE shared state instance

FILES CHANGED:
──────────────

✅ CREATED: src/providers/EntrepreneurProgressProvider.tsx (165 lines)
   - React Context provider
   - Wraps useEntrepreneurProgressState
   - Exports useEntrepreneurProgress from context

✅ CREATED: src/hooks/useEntrepreneurProgressState.ts (293 lines)
   - State management logic (extracted from old hook)
   - All useState and state update logic
   - Used by Provider to create shared state

✅ MODIFIED: src/hooks/useEntrepreneurProgress.ts
   - Changed from 267 lines → 5 lines
   - Now re-exports from provider
   - No breaking changes to API

✅ MODIFIED: src/app/_providers/RootProviders.tsx
   - Added EntrepreneurProgressProvider wrapper
   - 1 import, 1 component wrapping change
